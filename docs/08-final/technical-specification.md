# Техническое задание

> Техническое задание на разработку мобильного приложения для информационной системы таксопарка TaxiFleet Admin.

---

## 1. Общие сведения

### 1.1 Назначение документа

Настоящее техническое задание определяет требования к разработке мобильного приложения «TaxiFleet Admin» для управления таксопарком.

### 1.2 Назначение системы

Система предназначена для автоматизации процессов оперативного управления таксопарком:
- Учёт и управление водителями
- Учёт и управление автомобилями
- Создание и отслеживание заказов на перевозку
- Назначение водителей на заказы
- Мониторинг текущего состояния парка

### 1.3 Целевая аудитория

Администратор таксопарка — единственная роль пользователя системы.

## 4. Статусная модель

### 4.1 Статусы водителя (DriverStatus)

```
FREE ──→ BUSY ──→ FREE
  │                 ↑
  └→ UNAVAILABLE ──→┘
```

| Статус | Описание | Переходы |
|--------|----------|----------|
| FREE | Свободен, доступен для назначения | → BUSY (при назначении), → UNAVAILABLE |
| BUSY | Занят, выполняет заказ | → FREE (при завершении заказа) |
| UNAVAILABLE | Недоступен (отпуск, больничный) | → FREE |

### 4.2 Статусы автомобиля (CarStatus)

| Статус | Описание | Переходы |
|--------|----------|----------|
| AVAILABLE | Доступен для поездок | → ON_TRIP, → MAINTENANCE, → BROKEN |
| ON_TRIP | Выполняет заказ | → AVAILABLE |
| MAINTENANCE | На техническом обслуживании | → AVAILABLE |
| BROKEN | Неисправен | → MAINTENANCE, → AVAILABLE |

### 4.3 Статусы заказа (OrderStatus)

```
NEW ──→ ASSIGNED ──→ ON_WAY ──→ DONE
  │                              
  └──→ CANCELLED                 
```

| Статус | Описание | Переходы |
|--------|----------|----------|
| NEW | Новый заказ, ожидает назначения | → ASSIGNED, → CANCELLED |
| ASSIGNED | Назначен водитель | → ON_WAY, → CANCELLED |
| ON_WAY | Водитель в пути | → DONE |
| DONE | Заказ выполнен | (конечный) |
| CANCELLED | Заказ отменён | (конечный) |
